# Autonomous Workflow Builder (AWB)

OpenClaw creates its own tools. Self-expanding capabilities through workflow automation.

## Philosophy

**Build once, use forever. Automate everything.**

When you find yourself doing something twice, AWB builds a workflow for it.

### 4. Functions

Reusable code modules:
```python
# AWB detected: You calculate token scores frequently
# Built this function for reuse

def calculate_token_score(token_data):
    """
    AWB-generated function.
    Used 23 times across different projects.
    
    Scores: 0-100 (higher = better)
    Weights auto-optimized by ALOE.
    """
    weights = {
        'liquidity': 0.25,
        'holders': 0.20,
        'volume': 0.20,
        'narrative': 0.15,
        'contract': 0.20
    }
    
    score = sum(
        token_data[key] * weight
        for key, weight in weights.items()
    )
    return int(score)
```

## Workflow Types

### Type 1: One-Shot
Single execution, manual trigger:
```bash
"Research this token"
→ Execute once
→ Deliver report
→ Done
```

### Type 2: Scheduled
Runs on schedule:
```yaml
workflow: daily_market_scan
schedule: "0 */6 * * *"  # Every 6 hours
action: scan_market
output: send_report
```

### Type 3: Trigger-Based
Event-driven:
```yaml
workflow: new_token_alert
trigger: token_listed
condition:
  market_cap: < 1000000
  volume_24h: > 50000
action: generate_analysis
output: send_notification
```

### Type 4: Continuous
Always running:
```yaml
workflow: monitor_portfolio
mode: continuous
frequency: every_5_minutes
action: check_positions
alert_on: significant_change
```
